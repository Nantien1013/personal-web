# 啟動紀錄 — Laravel + Livewire 本機啟動

> 更新日期：2026-07-05
> 架構：PHP 8.3 + Laravel 13 + Livewire 3 + Tailwind v4，本機用 SQLite（免安裝資料庫伺服器）。
> 整個應用程式都在 `backend/` 資料夾內。

---

## 一、需求

- PHP 8.3 以上
- Composer 2.x
- Node.js 20 以上 + npm（建置前端資產用）

確認版本：

```bash
php -v
composer -V
node -v
```

## 二、安裝與啟動（本機 SQLite）

```bash
cd backend
composer install
npm install
cp .env.example .env
php artisan key:generate
php -r "file_exists('database/database.sqlite') || touch('database/database.sqlite');"
php artisan migrate --seed
npm run build
php artisan serve
```

瀏覽 <http://localhost:8000>。

開發時可改用資產監看（不必每次手動 build）：

```bash
# 視窗 1
php artisan serve
# 視窗 2
npm run dev
```

## 三、管理員登入

新增／編輯／刪除資料需要登入。`AdminSeeder` 會依 `.env` 建立一組管理員：

- 帳號：`admin@example.com`
- 密碼：`password123`

登入頁 <http://localhost:8000/login>。可在 seed 前用 `.env` 的 `ADMIN_EMAIL` / `ADMIN_PASSWORD`
修改（未開放註冊）。訪客可唯讀瀏覽全部頁面。

## 四、測試

```bash
cd backend
php artisan test
```

## 五、常見錯誤與解法

| 錯誤訊息 | 原因 | 解法 |
|---|---|---|
| `Vite manifest not found` | 尚未建置前端資產 | `npm run build`（或開著 `npm run dev`） |
| `No application encryption key` | 沒產生 APP_KEY | `php artisan key:generate` |
| `database disk image is malformed` / 找不到資料庫 | `database/database.sqlite` 不存在 | 先 `touch database/database.sqlite` 再 `php artisan migrate --seed` |
| 登入後仍不能編輯 | 帳號不是管理員 | 用 seed 出來的 admin 帳號登入 |
| 樣式跑掉／字體沒載入 | 資產未建置 | `npm run build` |

## 六、重新啟動

```bash
cd backend
php artisan serve   # 需要重建資料時： php artisan migrate:fresh --seed
```
