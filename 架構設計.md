# 個人網站 — 架構與資料表設計文件

> 最後更新：2026-04-22

---

## 一、專案概述

### 定位
- 個人入口網站，可分享給朋友瀏覽
- 不使用帳號登入系統（寫入 API 靠管理員 Token 保護）
- 模組化設計：新頁面可水平擴充，互不干擾

### 技術架構
| 層級 | 技術 |
| --- | --- |
| 前端 | HTML + CSS + JavaScript（Vanilla，無框架） |
| 後端 | Node.js + Express |
| 資料庫 | MySQL |
| 版本控制 | GitHub |
| 部署（前端靜態頁） | GitHub Pages |
| 部署（後端 + DB） | Railway 或 Render |
| 不使用 | Docker、WordPress、虛擬機 |

### 建置順序
1. 本機安裝 MySQL + Node.js + 視覺化工具（TablePlus / DBeaver）
2. 設計 MySQL 資料表結構
3. 用 Node.js + Express 寫後端 API
4. 動漫收藏頁前端接上 API（取代 localStorage）
5. 製作主頁 + 個人簡歷（靜態）
6. 部署到 Railway / GitHub Pages，取得公開網址
7. 開發論文單字庫（含自動查詢）

## 三、頁面內容概念

### 1. 主頁（index.html）

**畫面區塊**
- Hero：姓名、一句話定位、頭像
- 簡短自我介紹（2–3 行）
- 導覽卡片區：分「關於我 / 收藏 / 學習」三組
- 底部：GitHub、Email、LinkedIn 連結

**性質**：純靜態，不需資料庫

---

### 2. 個人簡歷（resume.html）

**畫面區塊**
- Header：姓名 + 聯絡資訊（Email、電話、地點、GitHub、LinkedIn）
- Education（學歷）
- Experience（經歷）
- Research & Project（研究與專案）
- Skills（技能，分類 tag）
- Certifications（證照）
- 下載 CV（PDF）按鈕

**增強想法**
- 專案卡片化，可點進去看細節
- 技能用顏色分類的 tag（資安紅 / ML 藍 / Programming 綠）
- 雙語切換（中 / 英）
- 時間軸 UI 呈現學歷與經歷

**性質**：純靜態，內容直接寫在 HTML

---

### 3. 收藏（collection.html）

支援 **動漫 + 漫畫** 兩種類型，同頁 Tab 切換。

**畫面區塊**
- 頁首：類型切換 Tab（全部 / 動漫 / 漫畫）+ 統計摘要（總數、已看完、觀看中、最愛）
- 操作列：新增、搜尋、篩選、排序、view 切換（卡片 / 表格）
- 主要列表：每張卡片顯示封面、標題、年份 + 番別 tag、類別 tags、星級、最愛 icon
- 新增 / 編輯 Modal：表單（欄位依類型動態切換）
- 空狀態 CTA

**篩選器**
- 類型（動漫 / 漫畫）
- 狀態（觀看中 / 已看完 / 計畫 / 擱置 / 放棄）
- 年份範圍
- 番別（冬 / 春 / 夏 / 秋）
- 類別（多選 tag，AND / OR 切換）
- 評分範圍
- 是否最愛

**番別定義**（日本動畫標準）
| 番別 | 月份 |
| --- | --- |
| 冬番 | 1–3 月 |
| 春番 | 4–6 月 |
| 夏番 | 7–9 月 |
| 秋番 | 10–12 月 |

**類別系統**（仿巴哈動畫瘋）
- **主題類**：戀愛、校園、科幻、奇幻、冒險、動作、戰鬥、運動、搞笑、治癒、音樂、美食、偵探、懸疑、恐怖、機甲、魔法、異世界、職場、歷史、戰爭、後宮、百合、萌系、日常
- **來源類**：原創、漫畫改編、輕小說改編、遊戲改編、小說改編
- **媒體類型**：TV 動畫、劇場版、OVA、ONA、特別篇、漫畫單行本、網路漫畫

### 4. 論文單字庫（vocabulary.html）

**核心功能**：輸入英文單字 → 自動查詢中文、詞性、例句、音標 → 使用者檢視 / 修改 → 存檔

**畫面區塊**
- 頁首：統計（總數、本週新增、待複習、平均熟悉度）
- 模式 Tab：瀏覽 / 新增 / 抽卡複習
- 瀏覽模式：搜尋 + 篩選 + 單字列表
- 新增模式：
  - 輸入英文單字 → debounce 500ms → 自動查詢
  - 自動填入：中文、詞性、KK 音標、例句、發音檔 URL
  - 所有欄位可編輯
  - **重複偵測**：已存在時跳警告（覆蓋 / 追加 / 取消）
- 抽卡模式：
  - 正面顯示英文 + 發音按鈕
  - 翻面看中文 + 例句
  - 自評「忘記 / 有點印象 / 記得 / 很熟」→ 更新熟悉度
  - 顯示今日進度

**抽卡演算法**（簡化 SM-2 / Leitner box）
- 新字 → 1 天後複習
- 答對 → 間隔翻倍（1 → 3 → 7 → 14 → 30 天）
- 答錯 → 重置為 1 天
- 「有點印象」→ 不翻倍，維持目前間隔
- 「很熟」→ 間隔 ×2.5

## 四、外部 API 整合（論文單字庫）

### API 選擇（免費、無需註冊）

**Free Dictionary API**
- URL：`https://api.dictionaryapi.dev/api/v2/entries/en/{word}`
- 回傳：詞性、英文釋義、例句、KK 音標、發音檔 URL
- 限制：僅英文釋義

**MyMemory Translation API**
- URL：`https://api.mymemory.translated.net/get?q={word}&langpair=en|zh-TW`
- 回傳：中文翻譯
- 限制：每日 5000 字（個人用足夠）、機器翻譯可能不精準

### 整合策略
1. 前端 debounce 輸入（停止打字 500ms 才查詢）
2. 前端呼叫自家後端 `GET /api/vocabulary/lookup?word=xxx`
3. 後端依序打 Free Dictionary → MyMemory
4. 合併結果回傳前端
5. 前端預填表單，使用者可修改
6. 查過的單字存 localStorage 快取，避免重複呼叫
7. API 失敗時顯示錯誤訊息，使用者手動填

### 為什麼要透過後端代理
- 避免 CORS 問題
- 統一做 rate limit 和 cache
- 未來換 API 不影響前端
- 不暴露呼叫行為

### 發音
- 優先用 Free Dictionary 回傳的 audio URL
- 備用方案：瀏覽器內建 `window.speechSynthesis`（永久免費）

## 七、安全性

### 管理員 Token 驗證
- 所有寫入類 API（POST / PUT / DELETE）都需要在 header 帶 `X-Admin-Token`
- Token 值存在 `.env` 的 `ADMIN_TOKEN`
- 後端 middleware 檢查，不符直接 401
- 讀取類 API 公開，任何人都能瀏覽

### 環境變數（`.env`）
```
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=
DB_NAME=personal_site
ADMIN_TOKEN=一串隨機字串
PORT=3000
ALLOWED_ORIGINS=https://<username>.github.io
```

### CORS
- 後端裝 `cors` 套件
- 只允許前端部署網域（從 `ALLOWED_ORIGINS` 讀）

### 其他
- `.env` 加進 `.gitignore`
- 提供 `.env.example` 作為範本
- 建議加 rate limit（`express-rate-limit`）防 API 濫用

## 八、部署策略

| 部分 | 平台 | 原因 |
| --- | --- | --- |
| 前端靜態頁（index、resume） | GitHub Pages | 免費、穩定 |
| 前端動態頁（collection、vocabulary） | GitHub Pages | 同上，透過 JS 打後端 API |
| 後端 API | Railway | 免費額度夠個人用 |
| MySQL | Railway 內建 | 跟後端同平台方便 |

### 備份策略
- Railway 免費方案資料庫可能被清，建議定期 `mysqldump`
- 匯出 `.sql` 備份到 Google Drive 或 repo 內的 `backups/`（加 `.gitignore`）
- 單字庫支援 CSV / JSON 匯出入，當第二層備份
